# Shared Cinema

一个单进程、低内存占用的共享观影服务。多个用户打开同一房间链接后，可以共同观看同一部本地上传的视频，播放/暂停/拖动进度实时同步，并支持绑定时间轴的弹幕。

## 特性

- FastAPI + 原生 WebSocket，同房间 2 到 10 人规模可用
- 视频上传使用可恢复的顺序分片上传，默认每片 8MB，单次写入块不超过 1MB
- 上传完成后优先生成 HLS 切片，移动端和慢网络优先走 HLS，浏览器不支持时再回退到 MP4
- 视频回退播放仍使用 `FileResponse`，天然支持 HTTP Range
- 房间状态在内存，弹幕历史持久化到 SQLite
- 原生 HTML/CSS/JS，无构建步骤，手机与桌面端都可直接打开
- Safari 走原生 HLS，其它浏览器通过页面内引入的 `hls.js` 播放 HLS；如果脚本不可用则回退到 MP4

## 环境要求

- Python 3.11+
- `ffmpeg` 和 `ffprobe` 可执行文件在 `PATH` 中可用

## 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
```

## 安装与启动

```bash
pip install -r requirements.txt
uvicorn app:app --host 0.0.0.0 --port 8000
```

然后打开：

```text
http://127.0.0.1:8000/
```

## 使用流程

1. 进入 `/` 点击“创建房间”
2. 页面会跳转到 `/room/{room_id}`
3. 点击“选择视频”上传本地文件；大文件会自动按分片上传，失败后会重试并尽量从已完成偏移继续
4. 上传完成后服务会生成 HLS 播放清单与切片
5. 把页面里的分享链接发给其他人
6. 任意人播放、暂停、拖动进度，其他人会在约 1 秒内同步
7. 点击“发弹幕”发送文本，自己的弹幕是金色，别人的弹幕是白色

## 设计说明

- 上传接口不是 `multipart/form-data`，而是前端先创建上传会话，再按固定分片顺序发送二进制数据。后端把分片直接落盘，并记录当前已完成偏移，浏览器重试时可以继续上传。
- 一个房间同一时刻只保留一个视频。上传新视频后，旧视频文件、旧 HLS 清单和旧分片都会被删除，房间播放状态重置为暂停、进度回到 0。
- 如果 `ffmpeg` 可用，上传完成后会生成 1 到 2 档 HLS 码率，用于移动端和慢网络播放；如果 HLS 生成失败，前端会继续使用原始 MP4 直链播放。
- 房间播放状态只保存在进程内。服务重启后，同步状态会丢失，但 SQLite 中的弹幕历史仍然保留。
- `data/` 是运行期目录，会自动生成。

## 常见说明

- 如果视频文件很大，请确保服务器磁盘空间足够
- 如果上传中断，重新选择同一个文件后会尝试从服务端已接收的偏移继续，而不是从 0 开始
- iPhone Safari 对视频格式更严格，建议优先上传 `MP4/M4V/MOV + H.264/AAC`；服务端会把原始文件保留为回退源，并额外生成 HLS 版本
- 服务默认无鉴权，拿到房间链接的人都能加入
- 建议在同一内网或小范围场景下使用
